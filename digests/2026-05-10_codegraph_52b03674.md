### TL;DR
CodeGraph is a local SQLite-based MCP tool for Claude Code that builds a semantic knowledge graph of a codebase to cut exploration from 94% fewer tool calls and 77% faster exploration, with the README claiming major wins across multiple real-world projects.

### Key Quote
“CodeGraph gives those agents a pre-indexed knowledge graph — symbol relationships, call graphs, and code structure.”

### Summary
- **What it is**
  - `@colbymchenry/codegraph` is a CLI/library/MCP server for indexing codebases into a semantic graph so Claude Code can answer exploration questions without repeatedly grepping and reading files.
  - Positioned as **100% local**: SQLite only, no API keys, no external services.

- **Main promise**
  - Supercharges Claude Code’s Explore agents by replacing repeated filesystem search/read operations with graph queries.
  - The README highlights headline performance claims: **94% fewer tool calls · 77% faster exploration · 100% local**.

- **Benchmark claims**
  - Tested on 6 codebases with Claude Opus 4.6 and Claude Code v2.1.91.
  - README claims an average of **92% fewer tool calls · 71% faster** across the set.
  - Individual examples cited:
    - **VS Code**: 3 calls vs 52 calls, 17s vs 1m 37s
    - **Excalidraw**: 3 calls vs 47 calls, 29s vs 1m 45s
    - **Claude Code (Python + Rust)**: 3 calls vs 40 calls, 39s vs 1m 8s
    - **Claude Code (Java)**: 1 call vs 26 calls, 19s vs 1m 22s
    - **Alamofire**: 3 calls vs 32 calls, 22s vs 1m 39s
    - **Swift Compiler**: 6 calls vs 37 calls, 35s vs 2m 8s
  - Key observations from the README:
    - Explore agents did not fall back to file reads when using CodeGraph.
    - Cross-language traversal worked.
    - The Swift benchmark traced a 9-step call chain.
    - The largest benchmarked codebase was the Swift Compiler repo with **25,874 files** and **272,898 nodes**.

- **How it works**
  - **Extraction**: tree-sitter parses code into ASTs; language-specific queries extract nodes and edges.
  - **Storage**: data lives in a local SQLite DB at `.codegraph/codegraph.db`, with FTS5 search.
  - **Resolution**: references are linked across calls, imports, inheritance, and framework patterns.
  - **Auto-sync**: watches files using native OS events with a 2-second debounce and incremental updates.

- **Key features**
  - Smart context building
  - Full-text symbol search
  - Callers/callees/impact analysis
  - Always-fresh auto-sync
  - Framework-aware route detection
  - Support for **19+ languages**:
    - TypeScript, JavaScript, Python, Go, Rust, Java, C#, PHP, Ruby, C, C++, Swift, Kotlin, Dart, Scala, Svelte, Vue, Liquid, Pascal/Delphi
  - Framework-aware route support spans Django, Flask, FastAPI, Express, Laravel, Rails, Spring, Gin/chi/gorilla/mux, Axum/actix/Rocket, ASP.NET, Vapor, React Router, and SvelteKit.

- **Framework-aware routes**
  - The graph creates `route` nodes and links them to handlers, so “who handles this URL?” becomes a graph query.
  - README gives concrete recognized shapes, including:
    - Django `path()`, `re_path()`, `url()`, `include()`
    - Flask `@app.route(...)`
    - FastAPI `@app.get(...)`, `@router.post(...)`
    - Express `app.get(...)`, `router.post(...)`
    - Laravel `Route::get()`, `Route::resource()`, `Controller@action`
    - Rails route syntax like `get '/x', to: 'users#index'`
    - Spring `@GetMapping`, `@PostMapping`, `@RequestMapping`
    - Gin/chi/gorilla/mux handler registration patterns
    - Axum/actix/Rocket `.route("/x", get(handler))`
    - ASP.NET `[HttpGet("/x")]`
    - Vapor `app.get("x", use: handler)`
    - React Router and SvelteKit route component nodes

- **Getting started**
  - Quick start:
    - `npx @colbymchenry/codegraph`
    - `cd your-project`
    - `codegraph init -i`
  - The installer:
    - offers global install
    - configures the MCP server in `~/.claude.json`
    - sets auto-allow permissions
    - adds instructions to `~/.claude/CLAUDE.md`
    - can optionally initialize the current project

- **Manual setup**
  - Global install:
    - `npm install -g @colbymchenry/codegraph`
  - MCP config snippet points Claude to:
    - `codegraph serve --mcp`
  - Optional auto-allow permissions are listed for:
    - `codegraph_search`
    - `codegraph_context`
    - `codegraph_callers`
    - `codegraph_callees`
    - `codegraph_impact`
    - `codegraph_node`
    - `codegraph_status`
    - `codegraph_files`

- **Claude Code usage model**
  - The README instructs that if `.codegraph/` exists:
    - do **not** call `codegraph_explore` or `codegraph_context` directly in the main session
    - spawn an Explore agent instead
    - use lightweight direct tools only for targeted lookup
  - If `.codegraph/` does not exist:
    - Claude should ask whether to initialize CodeGraph.

- **CLI reference**
  - `codegraph`
  - `codegraph install`
  - `codegraph init [path]`
  - `codegraph uninit [path]`
  - `codegraph index [path]`
  - `codegraph sync [path]`
  - `codegraph status [path]`
  - `codegraph query <search>`
  - `codegraph files [path]`
  - `codegraph context <task>`
  - `codegraph affected [files...]`
  - `codegraph serve --mcp`

- **`codegraph affected`**
  - Finds test files impacted by source changes via transitive dependency tracing.
  - Supports stdin from `git diff --name-only`, custom depth, custom test-file glob, and JSON/quiet output.
  - Includes a CI example that runs affected tests with Vitest.

- **Library usage**
  - Example API shows:
    - `CodeGraph.init()` / `CodeGraph.open()`
    - `indexAll()`
    - `searchNodes()`
    - `getCallers()`
    - `buildContext()`
    - `getImpactRadius()`
    - `watch()` / `unwatch()` / `close()`

- **Configuration**
  - `.codegraph/config.json` options include:
    - `languages`
    - `exclude`
    - `frameworks`
    - `maxFileSize`
    - `extractDocstrings`
    - `trackCallSites`

- **Troubleshooting**
  - Covers:
    - not initialized
    - slow indexing
    - WASM fallback / `database is locked`
    - MCP connection issues
    - missing symbols
  - Notes that `better-sqlite3` is optional and that the WASM fallback is slower.
