### TL;DR
This arXiv paper proposes **String Seed of Thought (SSoT)**, a prompting trick that makes LLMs better at **probabilistic instruction following** by first generating a random string and then transforming it into an answer, improving diversity and making output distributions closer to target probabilities.

### Key Quote
> “We propose SSoT, a simple prompting method that instructs an LLM to first output a random string to generate sufficient entropy.”

### Summary
- **Paper:** *String Seed of Thought: Prompting LLMs for Distribution-Faithful and Diverse Generation*
- **Venue/status:** arXiv CS.AI preprint; submitted **24 Oct 2025**, revised through **5 Feb 2026** (v3)
- **Core problem:** The authors define **Probabilistic Instruction Following (PIF)** as tasks where an LLM must sample answers according to a **specified probability distribution** over a fixed set of options.
- **Why this matters:** Standard LLM prompting tends to produce **biased, collapsed, or overly deterministic** outputs, which is bad for:
  - human-behavior simulation
  - content diversification
  - multiplayer games / non-deterministic agents
  - test-time scaling where diverse outputs matter
- **Method: String Seed of Thought (SSoT)**
  - Prompt the model to **first emit a random string**
  - Then **manipulate/extract randomness** from that string to derive the final answer
  - Goal: create enough entropy in the generation process while still respecting the desired answer constraints
- **Claimed results:**
  - SSoT **significantly improves PIF performance**
  - Performance is said to approach that of an **ideal pseudo-random number generator**
  - Benefits are not limited to closed-set choice tasks; on **NoveltyBench**, SSoT also improves **open-ended response diversity**
- **Main contribution:** A very simple prompting strategy that tries to use the model’s own generative process as a source of randomness, instead of fighting its tendency toward mode collapse.
