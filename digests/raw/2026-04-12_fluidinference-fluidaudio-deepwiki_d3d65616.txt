This document provides a high-level introduction to FluidAudio, a Swift SDK for fully local, low-latency audio AI on Apple platforms. It covers the SDK's core capabilities, architectural design, and design philosophy.
For installation instructions and initial setup, see Getting Started For detailed API documentation of individual modules, see Core Library For performance characteristics and benchmarking, see Performance and Benchmarks
FluidAudio is a Swift SDK that enables fully local audio AI processing on macOS and iOS devices. The SDK provides four primary AI capabilities:
- Automatic Speech Recognition (ASR): Transcription via Parakeet TDT, Qwen3, and Nemotron models supporting over 30 languages. README.md38-39 CLAUDE.md124-132 CLAUDE.md149-152
- Speaker Diarization: Speaker separation and identification using Pyannote segmentation, WeSpeaker embeddings, VBx clustering, and end-to-end Sortformer. README.md41-42 CLAUDE.md133 CLAUDE.md143
- Voice Activity Detection (VAD): Speech/silence classification using Silero VAD models. README.md43 CLAUDE.md135 CLAUDE.md145
- Text-to-Speech (TTS): Voice synthesis via PocketTTS (streaming, voice cloning) and Kokoro (high-quality, SSML and IPA support). README.md40 Documentation/TTS/Kokoro.md1-5 CLAUDE.md134
All inference operations are optimized for Apple's Neural Engine (ANE) via CoreML, minimizing CPU usage and power consumption while avoiding GPU/MPS entirely. README.md12-14 README.md44 Documentation/TTS/Kokoro.md98-100

FluidAudio provides two TTS engines catering to different quality and latency requirements. README.md40
| Feature | Kokoro (KokoroTtsManager ) | PocketTTS (PocketTtsManager ) |
|---|---|---|
| Synthesis | Parallel (All frames at once) | Autoregressive (Streaming) |
| Vocoder | Vocos | Mimi |
| Latency | Wait for full generation | ~80ms to first audio |
| Control | SSML, Custom Lexicon, IPA | Voice Cloning (1-30s) |
| Languages | 9 (EN, ES, FR, HI, IT, JA, PT, ZH) | English |
