## Multi-head attention

The previous article described how the whole Q, K, V thing works. But it did not describe _why_ this is better than, say, the LSTM-based [Sequence-to-Sequence model from 2014](https://arxiv.org/abs/1409.3215).

One benefit of the scaled dot-product attention mechanism is that it can be **parallelised**. What we’ve discussed so far is referred to as a single attention “head” in the paper. This “head” is able to (hopefully) capture how the words in the sentence are related to each other.

Rather than running one large attention head, the final model from the paper splits this into a set of h=8 smaller attention heads that can be operated at the same time. What does this mean? Well, recall the matrices Q, K and V that we used to compute self-attention before:

For our sentence “you should subscribe to my substack”, the expression Attention(Q, K, V) represents n=6 self-attention values, each corresponding to one of the sentence’s words.

For multi-head attention, we extend this by creating h=8 independent Query, Key and Value matrices, hence creating the 8 different self-attention values for each of the n=6 words in the sentence, so 48 self-attention values in total for our example. Mathematically, we can represent each attention head as

Interestingly, this parallelisation had a similar computational cost to that of the original single-head attention mechanism.

Wait, hang on: the multi-head version takes just as long to train? Then what was the point of doing all this?

By running multiple heads at once, we allow the transformer the ability to pick up on many different patterns in the training data simultaneously. What does this mean in context? For our exemplar sentence “you should subscribe to my substack”, we might want to pick out multiple intra-sentential relationships at once:

💬 The word ‘subscribe’ refers to the ‘substack’ in question. Also, ‘you’ are the one who is being advised to ‘subscribe’, while the word ‘my’ tells us who the ‘substack’ belongs to.

💬 One could extract the key call-to-action from the sentence as “you… should… subscribe”.

💬 Word types: ‘you’ is a personal pronoun, ‘should’ and ‘subscribe’ are verbs, ‘to’ is an adverb, ‘my’ is a determiner and ‘substack’ is a noun.

In an ideal scenario, each separate attention head would “attend” to each of these relationships. But remember, as always, that deep learning is inherently a black-box science. That is, we have no guarantee that the heads will pick up on these relationships in the ways we expect/hope. Additionally, this sentence is quite simple and short on its own, so it’s possible that multiple attention heads may attend to the same patterns, which is something we wouldn’t want in practice.

NB the paper’s appendix provides some visualisations of the patterns that were actually picked up by their model- I recommend checking it out if you’re interested.

## Comparison to other architectures

Ok fine, that explains the purpose of the multi-head approach. But it doesn’t yet explain why the transformer is better than previous architectures like, say, the LSTM-based [Sequence-to-Sequence model from 2014](https://arxiv.org/abs/1409.3215). What’s the big deal?

Well, for one thing, each LSTM requires the full output from the previous LSTM cell. Namely, we cannot process multiple components of the pipeline at the same time due to dependencies across cells. Conversely, each attention head can work to learn its own set of patterns, with the independent outputs concatenated after.

Another benefit of multi-head attention over traditional recurrent/convolutional techniques is the **reduction of necessary compute power for training**. For sequence length n and representation dimension d (i.e. the size of the embedding layer, size of encoder output, etc.), the paper notes the complexity of a self-attention layer as O(n 2· d), compared to that of a recurrent layer at O(n · d 2). If you haven’t seen this sort of thing before, it’s called **Big-O notation**. It provides a clean way for us to compare the runtimes of algorithms. At a high level, self-attention complexity increases quadratically with respect to sentence length, while recurrence complexity increases quadratically with respect to the representation dimension. In practice, representation dimension typically far exceeds sentence length, hence why self-attention is less computationally demanding.

## Packing it all up

That covers the main components of the multi-head attention mechanism, as well as a few other bits and pieces. Now for the usual summary:

📦 A key benefit of the scaled dot-product attention mechanism is that it can be parallelised. The 2017 researcher decomposed the overall single-head mechanism into a set of 8 smaller attention heads that could operate simultaneously.

📦 Although the multi-head mechanism of smaller heads had a similar computational cost to the larger single-head version, there is still benefit to the parallelised process. By running multiple heads at once, we allow the transformer the ability to pick up on many different patterns in the training data at once. The hope is that each head learns a unique pattern, the likes of which is not possible for a single-head mechanism.

📦 As for something like the 2014 model, each LSTM requires the full output from the previous LSTM cell. Namely, we cannot process multiple components of the pipeline at the same time due to dependencies across cells. Conversely, each attention head can work to learn its own set of patterns, with the independent outputs concatenated after.
