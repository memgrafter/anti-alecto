### TL;DR
This arXiv paper argues that off-the-shelf coding agents can outperform long-context LLMs and standard RAG by treating massive text corpora as file systems and using native tools like `grep`, `sed`, and Python scripts to search, parse, and aggregate information.

### Key Quote
“Rather than relying on latent attention or fixed retrieval, such agents can explicitly organize, filter, and transform text using executable programs.”

### Summary
- **Main claim:** long-context processing can be externalized from the model’s latent attention into an explicit environment where coding agents navigate files and run code.
- **Core setup:** the authors place corpora into directory structures (or single files for long documents) and give agents only a path plus a query; agents may use terminal commands, scripts, and intermediate files however they want.
- **Benchmarks evaluated:**
  - **BrowseComp-Plus**: multi-hop web/corpus QA over 100K documents
  - **Natural Questions (NQ)**: open-domain QA over a very large Wikipedia corpus
  - **LongBench-v2**: diverse long-context understanding tasks
  - **Oolong-Synthetic / Oolong-Real**: long-document aggregation and reasoning tasks
- **Reported results:** coding agents outperform published SOTA by **17.3% on average**, and set new SOTA on **4 of 5 benchmarks**.
- **Model/tool configurations tested:**
  - Codex v0.46.0 with GPT-5 backbone, no retriever
  - Codex + BM25
  - Codex + Gemini embeddings
  - Claude Code + BM25 on two benchmarks
- **Key finding on tools:** adding standard retrievers does **not** consistently help; in some cases it hurts because agents rely less on their own native exploration when a retriever is present.
- **Why it works, according to authors:**
  - **Native tool proficiency**: coding agents are good at executable search and manipulation, not just semantic querying
  - **File system familiarity**: hierarchical folders give them a strong inductive bias from coding tasks
- **Emergent strategies observed:**
  - **Iterative query refinement** for multi-hop retrieval tasks (especially BrowseComp-Plus)
  - **Programmatic aggregation** for analytical/document-wide tasks (especially Oolong)
  - **Direct inference / low-tool use** for some LongBench tasks where the base model’s long-context ability is enough
- **Ablation on structure:** folder-based corpus organization beats a single-file JSON-style corpus; agents use more `sed`/line-based slicing and less brute-force scanning.
- **Cost analysis:** coding agents cost more than simple RAG, but are competitive with or cheaper than other strong methods while performing much better.
- **Interpretation:** the paper’s broader thesis is that as coding-capable models improve, “coding” and “general text processing” begin to look like the same capability when text is structured like code.
