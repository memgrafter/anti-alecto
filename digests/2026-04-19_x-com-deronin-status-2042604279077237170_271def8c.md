### TL;DR
The thread lays out a “content skill graph” workflow: a folder of interconnected markdown files plus an AI agent can turn one idea into platform-native posts for X, LinkedIn, Instagram, TikTok, YouTube, Threads, Facebook, and a newsletter without manually rewriting each one.

### Key Quote
“It's literally like compound interest but for your content system”

### Summary
- The core claim is that most AI content workflows fail because they lack context; instead of prompting a chatbot from scratch each time, the author recommends building a persistent knowledge system made of `.md` files linked with `[[wikilinks]]`.
- This system is called a **skill graph**: a folder of interconnected markdown “knowledge nodes” that encode:
  - brand identity
  - audience segments
  - platform rules
  - voice/tone
  - hook formulas
  - repurposing logic
  - scheduling rules
  - content format definitions
- The suggested folder structure includes 17 files across 4 folders:
  - `index.md`
  - `platforms/` for X, LinkedIn, Instagram, TikTok, YouTube, Threads, Facebook, newsletter
  - `voice/` for brand voice and platform tone
  - `engine/` for hooks, repurposing, scheduling, content types
  - `audience/` for builders vs casual readers
- `index.md` is framed as the command center, not a table of contents. It defines:
  - the brand identity
  - the node map
  - execution instructions for the AI agent
- Each platform file contains platform-specific rules and posting strategy:
  - **X**: short, hook-driven, casual, lowercased, contrarian/proof hooks, 280-char focus
  - **LinkedIn**: professional but human, narrative-driven, 1,300–2,000 chars, first line matters
  - **Instagram**: visual-first carousels and Reels, bold first slide, captions support visuals, hashtags work
  - **TikTok**: raw video, hook in first 2 seconds, screen recordings and voiceover work well
  - **YouTube**: SEO-driven, 8–12 minute tutorials, title/thumbnail are critical
  - **Threads**: relaxed, conversational, opinion-driven, similar to X but softer
  - **Facebook**: community-oriented, longer captions, discussion prompts
  - **Newsletter**: email-based, personal, 1,000–2,000 words, plain-text feel
- `brand-voice.md` defines the underlying personality across all platforms:
  - casual but credible
  - direct and personal
  - anti-fluff
  - coaching-oriented
  - proof- and numbers-driven
- `platform-tone.md` explains how the same voice changes by platform:
  - X = most casual
  - LinkedIn = professional but human
  - Instagram = simplified and visual
  - TikTok = energetic and spoken
  - YouTube = structured and educational
  - Newsletter = intimate and reflective
  - Threads = relaxed and opinionated
  - Facebook = warm and community-oriented
- `hooks.md` gives reusable opener formulas:
  - playbook hook
  - proof hook
  - contrarian hook
  - replacement hook
  - discovery hook
  - behind-the-scenes hook
- `repurpose.md` defines the production chain:
  1. write for X first
  2. expand for LinkedIn
  3. convert to Instagram carousel
  4. condense into TikTok
  5. expand into YouTube tutorial
  6. deepen into newsletter
  7. relax into Threads
  8. adapt into Facebook discussion post
- The key principle is **“rethinking, not reformatting”**:
  - each platform version should be a distinct native piece, not the same text copied around
- The author recommends three implementation paths:
  - **Claude Projects** as the preferred setup
  - **paste context manually** into any chat tool
  - **Cursor / Claude Code** for a more autonomous local-file workflow
- The post gives a concrete example topic: “How I use AI to manage 10 social media accounts,” and shows how each platform would angle it differently.
- It ends with a call to action to like and repost, plus a promise of a future “Research Engine” article/video if the post reaches 2,000 likes.
