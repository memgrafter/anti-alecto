### TL;DR
JakeKAllDay argues that high-effort reasoning modes in frontier LLMs deliver only small benchmark gains for dramatically more tokens and cost, so most users should start with low/medium effort and only escalate when a task can’t be broken into smaller parts.

### Key Quote
“low or medium reasoning effort provide nearly all of the intrinsic performance of a model; high/xhigh/max configurations are extremely poor value”

### Summary
- **Core thesis:** Reasoning models are useful, but pushing them to high/xhigh/max effort is usually a poor tradeoff because token consumption rises much faster than performance.
- **Main claim:** Across recent frontier models, the last slice of benchmark performance is very expensive:
  - A **<20% performance boost** can drive about a **5x increase in output tokens and cost** for GPT-5.
  - The author argues this resembles **overclocking a processor**: great for edge cases, not representative of typical efficiency.
- **Evidence and examples:**
  - Early reasoning-effort curves for **GPT-5** showed a clear relationship between more thinking tokens and better scores.
  - The same diminishing-return pattern appears in **GPT 5.5**, though it is noted as more token-efficient than some other frontier models.
  - Recent frontier jumps in token use are highlighted:
    - **Claude Sonnet 4 → 4.6:** from **55M tokens to 200M** on the same benchmark.
    - **Opus 4.5 → 4.6:** **72M to 160M** tokens.
    - **GPT-5.4 xhigh:** up to **120M tokens** on the cited benchmark.
    - In one comparison, **Gemini 3.1** maintained similar token usage to Gemini 3 while improving performance.
    - Some open-source/frontier models like **Kimi K2.5 + GLM 5** reportedly used **10–35% fewer tokens** than predecessors.
- **Benchmark analysis:**  
  The author fits a saturation curve to **Artificial Analysis Intelligence Index v4** data (~10,500 questions) to estimate the relationship between thinking tokens and performance.
  - For the broader Q4/Q1 frontier, the fit is described as:
    - `Intelligence(x) = 33.18 × (1 − e−x / 20,123) + 23.82 (R² = 0.87)`
  - For April 2026 frontier data:
    - `Intelligence(x) = 24.14 × (1 − e^(−x / 9,450)) + 35.86 (R² = 0.90)`
  - For a GPT 5.5 over-fit:
    - `Intelligence(x) = 28.11 × (1 − e^(−x / 6,764)) + 31.89 (R² = 0.98)`
- **Important token-efficiency takeaway:**
  - In a typical **10-task packet**, about **90% of the performance gain** is achieved by roughly **46,000 tokens** in one analysis.
  - Reaching **99% frontier performance** requires about **93,000 tokens**.
  - In the GPT 5.5 update, **90% of the gain** is reached at about **15,600 tokens**, while **99%** requires around **31,100 tokens**.
  - The last **1%** of performance can cost roughly **50% more**.
- **Commercial/strategic interpretation:**
  - Providers have incentives to offer high-effort settings because they increase revenue and help models top benchmark charts.
  - The author suggests some “cheaper” models are effectively more expensive in practice once their large token use is included.
  - The piece also argues that benchmark results can be misleading because they often use the model’s **highest effort configuration**, not a standardized one.
- **Practical recommendations:**
  1. Start with **low or medium reasoning effort**.
  2. Use **non-thinking / miniature models** for simple tasks or sub-agents.
  3. Escalate to **high/max** only when results are inadequate and the task can’t be decomposed further.
- **Update note (4-23-26):**
