### TL;DR
A systematic survey of “deep research” systems that combine LLMs with tools for autonomous multi-source information gathering, organizing them into a three-stage roadmap, four core system components, and several training/evaluation challenges.

### Key Quote
“Query Planning refers to the process of transforming a complex and logically intricate question into a structured sequence of executable sub-queries”

### Summary
- **Scope and purpose**
  - Surveys **40+ deep research systems** for LLMs that use external tools to support **autonomous, multi-source, verifiable knowledge acquisition and synthesis**.
  - Contrasts deep research with standard **RAG**, emphasizing that DR systems go beyond static retrieval into dynamic, tool-augmented workflows.

- **Main framework**
  - Formalizes DR as a **three-stage evolution**:
    - **Agentic Search**
    - **Integrated Research**
    - **Full-stack AI Scientist**
  - Identifies **four core components**:
    - **Query planning**
    - **Information acquisition**
    - **Memory management**
    - **Answer generation**

- **Query planning**
  - Breaks complex questions into sub-queries using:
    - **Parallel planning**
    - **Sequential planning**
    - **Tree-based planning**
  - Tree-based planning recursively combines parallel and sequential structure.

- **Information acquisition / retrieval**
  - Covers retrieval tools and filtering strategies.
  - Highlights **adaptive retrieval triggering** based on confidence/uncertainty signals.
  - Four confidence paradigms are noted:
    - Probabilistic
    - Consistency-based
    - Internal state probing
    - Verbalized confidence
  - Examples mentioned include **Self-RAG** and **Search-R1**.

- **Memory management**
  - Defines a memory lifecycle with four operations:
    - **Consolidation**
    - **Indexing**
    - **Updating**
    - **Forgetting**
  - Distinguishes **unstructured** and **structured** consolidation approaches.
  - Emphasizes the need for explicit memory beyond the context window for long-horizon coherence.

- **Optimization / training**
  - Categorizes learning methods into:
    - **Workflow prompting**
    - **Supervised fine-tuning (SFT)**
    - **End-to-end reinforcement learning (RL)**
  - Notes that SFT helps with cold start but reduces exploration entropy, while RL can discover better behaviors but is harder to stabilize.
  - Mentions **multi-stage training pipelines** and algorithms like **PPO/GRPO**.

- **Benchmarks and evaluation**
  - Reviews benchmarks for **complex queries** and **interactive environments**.
  - Highlights the lack of a unified evaluation protocol for what counts as “deep research.”
  - Flags the difficulty of distinguishing **generative novelty** from **hallucination**.

- **Core mechanisms highlighted by the digest**
  - **Hierarchical query decomposition**: split complex questions into executable pieces.
  - **Adaptive retrieval via confidence**: retrieve only when the model detects uncertainty.
  - **Memory lifecycle management**: consolidate, index, update, and forget over long horizons.

- **Open challenges**
  - Better **cold-start methods** that preserve exploration during RL.
  - More **fine-grained reward design** to optimize retrieval timing.
  - Better evaluation frameworks for **novelty vs. hallucination**.
  - More concrete transition criteria between the three stages of DR.

- **Practical implications**
  - Useful as a roadmap for designing research agents that do iterative search, maintain long-term context, and synthesize evidence with verification.
  - The digest proposes experiment ideas using **HotpotQA** and **AutoSurvey**-style tasks to test planning, retrieval, and memory strategies.
