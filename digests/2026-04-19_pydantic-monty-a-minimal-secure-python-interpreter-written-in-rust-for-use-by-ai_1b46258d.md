### TL;DR
Monty is an experimental Rust-based Python interpreter from Pydantic designed to safely and extremely quickly run LLM-generated code with strict host isolation, snapshotting, and resource limits.

### Key Quote
"To run code written by agents."

### Summary
- **What it is**
  - Monty is a **minimal, secure Python interpreter written in Rust** for AI/agent use.
  - The project is explicitly labeled **experimental** and “not ready for the prime time.”
  - It is intended to avoid the latency and complexity of full container sandboxes when running code generated by LLMs.

- **Core design goals**
  - Run a **reasonable subset of Python**, not full CPython.
  - **Block access** to host filesystem, environment variables, and network by default.
  - Allow only **explicitly provided host functions** to be called.
  - Support **type checking** with modern Python type hints; `ty` is bundled in the single binary.
  - Support **snapshotting** execution state to bytes so code can be paused, stored, and resumed later.
  - Track and limit **memory usage, allocations, stack depth, and execution time**.
  - Return captured **stdout/stderr** to the caller.
  - Be usable from **Rust, Python, or JavaScript**.

- **What it can do**
  - Run a useful subset of Python for agent workflows.
  - Execute code with external host function calls controlled by the developer.
  - Resume execution iteratively via `start()` / `resume()`.
  - Serialize both parsed programs and execution snapshots.
  - Run sync or async host callbacks.
  - Use a limited stdlib subset: `sys`, `os`, `typing`, `asyncio`, `re`, `datetime`, `json`, and `dataclasses` (noted as “soon”).

- **What it cannot do**
  - Use most of the standard library.
  - Use third-party Python libraries.
  - Define classes yet.
  - Use `match` statements yet.

- **Intended use case**
  - The repo frames Monty as a tool for **agent-written code execution**, especially as an alternative to traditional tool-calling flows.
  - It is positioned as a future implementation detail for **Pydantic AI codemode**.

- **Python usage example**
  - Install with:
    - `uv add pydantic-monty`
    - or `pip install pydantic-monty`
  - Example shows:
    - Creating `Monty(code, inputs=[...], type_check=True, type_check_stubs=...)`
    - Passing an async external function `call_llm`
    - Running with `await m.run_async(...)`
  - Another example shows iterative external calls:
    - `m.start(...)` pauses at `fetch(url)`
    - `result.resume({'return_value': ...})` continues execution
  - Serialization examples show:
    - `m.dump()` / `Monty.load(data)` for parsed code
    - `progress.dump()` / `load_snapshot(state)` for mid-flight state

- **Rust usage example**
  - Uses `monty::{MontyRun, MontyObject, NoLimitTracker, PrintWriter}`
  - Demonstrates running a recursive `fib(n)` function and getting `55` for input `10`
  - Shows Rust-side serialization via `dump()` / `load()`

- **Pydantic AI integration**
  - The README includes a larger example where Monty powers a **CodeModeToolset** inside a Pydantic AI agent.
  - The example uses a weather toolset with functions like:
    - `get_lat_lng`
    - `get_temp`
    - `get_weather_description`
  - The agent can compare weather across cities by having the LLM write code that invokes tools as functions.

- **Community and ecosystem**
  - A **Go binding** exists: `gomonty`
  - The repo invites Slack participation and points to CI, CodSpeed, Codecov, PyPI, and license badges.

- **Alternatives section**
  - The README contains a long comparison against:
    - **Docker**
    - **Pyodide**
    - **starlark-rust**
    - **WASI / Wasmer**
    - **sandboxing services** like Daytona, E2B, Modal
    - **YOLO Python**
  - The comparison focuses on:
    - language completeness
    - security
    - startup latency
    - setup complexity
    - file mounting
    - snapshotting
  - Monty’s claimed advantages are:
    - microsecond startup
    - strict security controls
    - easy setup
    - easy snapshotting
  - The tradeoff is reduced language completeness and no third-party libraries.
