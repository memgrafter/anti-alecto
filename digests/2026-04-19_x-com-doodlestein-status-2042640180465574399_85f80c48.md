### TL;DR
The post argues that agent “skills” are far more powerful than simple Markdown docs because they can bundle large reference corpora, scripts, and subagents to drive practical security audits of SaaS codebases.

### Key Quote
“People who dismiss agent skills as ‘just markdown files’ are really missing out on what’s possible.”

### Summary
- The author pushes back on the idea that agent skills are merely Markdown files, saying they can include:
  - scripts
  - subagents
  - large reference bundles
  - workflow guidance for agents
- Main framing: skills are a **delivery mechanism** for automated workflows:
  - they supply a large amount of reference material
  - they do so in a way that is “agent-ergonomic” and “agent-intuitive”
- The concrete example is a new skill on the author’s site, **jeffreys-skills.md**, called **`security-audit-for-saas`**.
- The skill is described as being based on **many months of the author’s actual agent coding sessions** across multiple SaaS projects.
- Scale of the skill:
  - **90 files**
  - **888 KB** total
- It contains broad security knowledge for modern SaaS apps, including:
  - **Supabase**
  - **Stripe**
  - **PayPal APIs**
  - and other SaaS-related security concerns
- The author argues that even if frontier models already “contain” this knowledge in their weights, that is not enough:
  - the models still need guidance
  - they need coaching/coaxing to apply knowledge effectively in practice
- The skill is presented as reusable and broad enough to be applied multiple times across different parts of a codebase:
  - different passes
  - different analytical lenses
  - different traversal paths
- The author highlights modern agent harnesses like **Claude Code** and **Codex** as especially well-suited to this kind of **massively parallel** analysis using **subagents**.
- A key benefit claimed for skills as a standard:
  - the same skills can work across popular harnesses and models
  - results can still differ depending on the harness/model driving them
  - this reduces vendor lock-in
  - it provides “diversity of thought for free”
- Security impact claim:
  - for a reasonably large SaaS project, this skill is likely to uncover:
    - at least a few **critical vulnerabilities**
    - plus many smaller edge-case issues worth fixing
- The post ties this to recent Anthropic disclosures about a **new Mythos model**, which reportedly chained together multiple individually minor issues into devastating exploits.
- Closing implication:
  - systematic discovery and fixing of even small issues matters more than ever, because attackers can combine weak points into serious attacks.
