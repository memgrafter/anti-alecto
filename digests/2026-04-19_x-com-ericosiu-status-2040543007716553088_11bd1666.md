### TL;DR
Eric Siu argues that Jack Dorsey’s “From Hierarchy to Intelligence” is the blueprint for AI-native companies, then maps it onto a real company’s four-month experiment with agents, a shared company memory (“Single Brain”), and AI-driven workflows—while emphasizing the messy security, coordination, and implementation problems.

### Key Quote
“Month 1 of this system was terrible. Agents hallucinated. Data was wrong. Automations broke at 3am.”

### Summary
- This is a long X thread reacting to Jack Dorsey and Roelof Botha’s essay, **“From Hierarchy to Intelligence”** (said here to have dropped **April 1** and reached **5 million views in 48 hours**).
- The author claims their company has spent **4 months** rebuilding around AI agents and says they already have a working version of the architecture Dorsey described.
- The thread presents **four layers** of an AI-native organization:
  - **Layer 1: Capabilities** — the raw AI tools/models.
  - **Layer 2: World Model** — the company’s living memory / data structure.
  - **Layer 3: Intelligence Layer** — the decision-making agent layer.
  - **Layer 4: Surfaces** — where humans interact with the system.
- It argues that the biggest misconception is that the “world model” is the AI itself; instead, it is the **data infrastructure** that gives AI context about the business.

#### Their internal stack / architecture
- The hardware/software stack is shown in a code block, but the capture is **incomplete/truncated**. What’s visible includes:
  - **Mac Mini M4 (32GB)** — running **Alfred: CEO agent**, **crons**, and message routing.
  - **DGX Spark GB10 (128GB)** — local inference, team agent hosting, “World Brain vector DB.”
  - **Mac Studio Ultra** — client agent fleets and overflow compute.
- The thread says moving from cloud API endpoints to **local inference** reduces costs by roughly **70%**.
- It mentions **Gemma 4** and another truncated reference, but the sentence is cut off, so the full model comparison is not reliable from this scrape.

#### “Single Brain” / world model
- Their shared memory layer is called **Single Brain**.
- It is described as a **unified vector database** that ingests company data **every 15 minutes** from:
  - Slack
  - CRM
  - Gong call transcripts
  - Google Analytics
  - Search Console
  - client deliverables
  - meeting notes
  - financial data
- Every agent queries this same brain so decisions are informed by the full business context.
- The thread emphasizes that this takes **months** to build because the value comes from accumulating proprietary company data over time.

#### Agent layer
- The author lists specialized agents:
  - **Alfred** — CEO operations, calendar, email triage, task coordination, strategic analysis.
  - **Arrow** — sales.
  - **Oracle** — SEO.
  - **Flash** — organic social content.
  - **Cyborg** — recruiting.
  - **The World Agent** — coordinates across all agents.
- Two automated systems are also mentioned:
  - **AutoResearch** — continuous pattern mining.
  - **AutoGrowth** — automatic A/B experiments.
- The org structure is described as moving from a traditional hierarchy to:
  - CEO
  - World Agent
  - specialized agents
  - temporary DRI teams
  - human IC specialists
- One of the embedded code blocks is visibly **cut off** in the agent list (“recruit…”), so the org chart in the capture is not fully intact.

#### What breaks in practice
- The thread’s most useful operational section is the list of failure modes:
  - agents conflict when they operate on the same data
  - one agent may schedule something another agent already claimed
  - sales may promise timelines unsupported by SEO/ops data
- The author says they had to build:
  - a **conflict resolution system**
  - a **security layer** called **NemoClaw**
  - **kernel-level sandboxing**
  - multi-tier permissions:
    - data everyone can see
    - role-restricted data
    - CEO/org-brain-only data
- It includes a concrete warning: **one agent nearly emailed a client’s financial data to the wrong contact**, prompting a stronger security posture.
- The author explicitly says to **plan for security on day one**.

#### DRI system
- The thread says the **DRI (Directly Responsible Individual)** concept is borrowed from Dorsey’s essay.
- In this implementation, a DRI can be a **temporary agent team**:
  - given a goal
  - run for **90 days**
  - either hit the target or dissolve
  - return learnings to the World Brain
- The author says this is the most immediately copyable part of the framework:
  - pick top 3 goals
  - create a DRI for each
  - assign agents
  - set a deadline
  - measure outcomes

#### Compounding / timeline
- The thread gives a three-month progression:
  - **Month 1**: terrible; hallucinations, bad data, broken automations
  - **Month 2**: some useful pattern detection begins
  - **Month 3**: the flywheel starts turning; outputs improve as more data accumulates
- It argues the real moat is not the model or the tools, but the **compounding company-specific data loop**.
- It says companies with active CRM and Slack usage will compound faster than sparse-data companies.

#### Business/product angle
- The author claims this internal system becomes a **product/service offering** for clients.
- Their agency model shifts from:
  - selling services
  - to selling the **intelligence layer** that makes services more effective
- The pitch to agencies/consultancies:
  - build internally first
  - use the system as your differentiator
  - sell what you’ve already proven works

#### Strong opinion / thesis
- The thread argues most companies will **not** implement this because:
  - org-chart changes are threatening
  - upfront investment is real
  - the first months are painful
  - boards and middle managers will resist
- But companies that do will operate at a **fundamentally different speed**, described hyperbolically as “1000% faster.”
