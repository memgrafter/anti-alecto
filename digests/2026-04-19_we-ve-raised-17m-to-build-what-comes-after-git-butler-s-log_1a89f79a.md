### TL;DR
GitButler announces a $17M Series A led by a16z to build a new version-control layer for modern, trunk-based, human-and-agent software collaboration beyond traditional Git.

### Key Quote
“We’re not building some ‘better git’.”

### Summary
- GitButler says it has raised a **$17M Series A** led by **a16z**, with continued support from seed investors **Fly Ventures** and **A Capital**.
- The post frames this as part of a longer personal and technical story from **Scott Chacon**:
  - Co-founder of **GitHub**
  - Has watched **Git** evolve over ~15 years from a niche developer tool to core software infrastructure
- Core thesis:
  - **Git is increasingly mismatched to modern development**, especially with:
    - short-lived branches / **GitHub Flow**
    - trunk-based workflows
    - **AI agents** participating in code changes
  - The hard problem is no longer just generating code, but **organizing, reviewing, and integrating change** without chaos.
- GitButler’s pitch is that development tools should reduce friction in collaboration and context switching across:
  - tools
  - people
  - people and agents
- The company recently released a **technical preview of the GitButler CLI**:
  - designed for **humans, agents, and scripting**
  - supports **stacked branches**
  - intended to help users **multitask**
  - lets users **control and organize changes**
  - emphasizes easy **undo**
  - drops into **existing Git projects**
- The broader vision goes beyond a CLI:
  - revive and extend the idea of **“Social Coding”**
  - make coding genuinely collaborative, not just “PR-based”
  - store and connect more of the surrounding context that is currently lost:
    - commit messages
    - PR descriptions
    - agent interactions
    - related conversations
- The post argues current team workflows are still fragmented:
  - issues and kanban boards remain separate
  - chat happens in external tools
  - useful context is lost outside Git history
  - merge conflicts are often discovered too late
- Future vision described in the post:
  - version control that helps craft **logical, beautiful changes**
  - tooling that surfaces **possible merge conflicts early**
  - support for **stacked work across teammates**
  - agents that understand not only their own work but the team’s live context
- Closing claim:
  - GitButler is building **“the infrastructure for how software gets built next”**, not merely an improved Git.
