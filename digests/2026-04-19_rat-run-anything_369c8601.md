### TL;DR
rat is a tiny MCP-based runtime wrapper that turns language REPLs into shared, cross-app sessions so you can load state once in a terminal and access the same variables from Claude Desktop, VS Code, Cursor, or any MCP client.

### Key Quote
“Your terminal, Claude, VS Code, and Cursor all see the same variables.”

### Summary
- **What it is**
  - rat (“Run AnyThing”) is presented as **one binary** that wraps language runtimes as **MCP servers**.
  - It lets you start a REPL/kernel in your terminal and then connect to that same session from other tools.
  - The central promise is a **shared namespace**: the same variables and session state are visible across terminal, desktop apps, and MCP clients.

- **Core workflow**
  - Install is pitched as extremely simple: **“One curl. One install command. Done.”**
  - Example use case: start a **Python kernel** in the terminal, load a dataframe, then ask Claude Desktop to analyze it without reloading or glue code.
  - It emphasizes **state continuity**: “Same variables. Same session.”

- **Feature set**
  - It claims **full IPython** support, including:
    - syntax highlighting
    - tab completion
    - magic commands
  - It says there are only **four tools** in the API:
    - **run** — execute code or provide input
    - **look** — inspect variables, complete code, see state
    - **tail** — view recent activity log
    - **ctl** — reset, cancel, restart
  - These same tools apply across all supported languages and client types.

- **Why it matters / positioning**
  - The pitch is that rat removes friction between:
    - terminal workflows
    - LLM assistants like Claude Desktop
    - editors like VS Code and Cursor
    - notebooks or other MCP-compatible apps
  - It highlights **automatic isolation by project directory**:
    - e.g. `rat py` in `~/project-a` is separate from `~/project-b`
    - no manual configuration is needed.
  - It claims support for **Python, R, bash, and “pi agent”** built in.
  - It also claims extensibility: any language can be added with **a YAML file and a kernel script**, without writing Go code.

- **Main takeaway**
  - The page is essentially a product landing page for a developer tool that makes interactive programming sessions portable across apps by exposing them as MCP servers.
