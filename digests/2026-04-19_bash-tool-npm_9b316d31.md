### TL;DR
`bash-tool` is a TypeScript package that gives AI agents a sandboxed bash environment plus file read/write tools, with optional Vercel Sandbox integration, command interception hooks, and experimental “skills” support.

### Key Quote
“Generic bash tool for AI agents, compatible with AI SDK.”

### Summary
- **What it is**
  - `bash-tool` is an npm package for creating a bash-based toolset for AI agents.
  - It is designed to work with the **AI SDK** and supports agent workflows that need command execution and file manipulation.
- **Installation**
  - Basic install:
    - `npm install bash-tool just-bash`
  - For full VM support, the docs recommend installing **`@vercel/sandbox`** instead of relying only on `just-bash`.
- **Basic usage**
  - Import `createBashTool` from `bash-tool`.
  - Create tools from an initial file set:
    - Example files shown:
      - `src/index.ts` with `export const hello = 'world';`
      - `package.json` with `{"name": "my-project"}`
  - Use the returned `tools` with `ToolLoopAgent` from `ai`.
  - The sample agent is configured with:
    - `model: yourModel`
    - `tools`
    - `stopWhen: stepCountIs(20)`
  - The example prompt is: “Analyze the project and create a summary report”.
- **Available tools**
  - **`bash`**
    - Executes bash commands in the sandbox environment.
    - Input: `command` string.
    - Returns: `stdout`, `stderr`, `exitCode`.
    - The docs say this may be the only tool needed for analysis agents.
  - **`readFile`**
    - Reads file contents from the sandbox.
    - Input: `path`.
    - Returns: `content`.
  - **`writeFile`**
    - Writes content to a sandbox file and creates parent directories if needed.
    - Input: `path`, `content`.
    - Returns: `success` boolean.
- **Advanced usage / integration options**
  - **Upload a local directory**
    - `uploadDirectory: { source: "./my-project", include: "**/*.{ts,json}" }`
    - The glob filter is optional.
  - **Use `@vercel/sandbox` for full VM support**
    - Create a sandbox with `Sandbox.create()`
    - Pass it into `createBashTool({ sandbox, files: ... })`
    - Files are written to `./workspace` by default.
  - **Persistent sandbox across serverless invocations**
    - Use `Sandbox.get({ sandboxId })` to reconnect to a previously created sandbox.
    - The docs emphasize that prior files and state are preserved.
  - **Use a custom `just-bash` instance**
    - Example creates `new Bash({ cwd: "/app" })`
    - Pass `destination: "/app"` to `createBashTool`.
  - **Intercept bash commands**
    - `onBeforeBashCall` can inspect or replace a command before execution.
    - Example blocks dangerous commands like `rm -rf`.
    - `onAfterBashCall` can inspect or modify results after execution.
    - Example trims `stdout`.
  - **Custom sandbox implementation**
    - The package exposes a `Sandbox` interface.
    - Example custom sandbox implements:
      - `executeCommand(command)`
      - `readFile(path)`
      - `writeFiles(files)`
- **Experimental skills support**
  - The package includes experimental support for **skills**, described as modular capabilities that extend agent functionality.
  - A skill is a directory containing a `SKILL.md` file and optional scripts.
  - Workflow shown:
    - Use `experimental_createSkillTool` to discover skills in `./skills`
    - Get back:
      - `skill`
      - `files`
      - `instructions`
    - Pass `files` and `extraInstructions` into `createBashTool`
    - Combine `skill` with the bash tools in `ToolLoopAgent`
  - The docs mention a public directory of skills via **Skills.sh** and include a sample directory structure with `csv` and `text` skills.
- **AI agent instructions**
  - There is a note that additional guidance for agents is in `AGENTS.md`:
    - `cat node_modules/bash-tool/dist/AGENTS.md`
- **License**
  - MIT
