### TL;DR
This 2025 arXiv paper reports that experienced software developers use AI coding agents as productivity aids but still keep tight control over design and implementation to preserve software quality.

### Key Quote
“experienced developers value agents as a productivity boost, they retain their agency in software design and implementation out of insistence on fundamental software quality attributes”

### Summary
- **Topic and scope**
  - Paper: *Professional Software Developers Don't Vibe, They Control: AI Agent Use for Coding in 2025*
  - Category: Computer Science > Software Engineering
  - Submitted on **16 Dec 2025**
  - Focuses on how **experienced developers** actually use AI agents in professional software development.

- **Research question**
  - Investigates what roles AI agents play in real-world software work.
  - Examines:
    - **motivations** for using agents
    - **strategies** developers use
    - **task suitability**
    - **sentiments** toward agent use

- **Method**
  - **Field observations (N=13)**
  - **Qualitative surveys (N=99)**
  - The paper is based on empirical study of developer behavior rather than just conceptual argument.

- **Main findings**
  - Developers see AI agents as a **productivity boost**.
  - They do **not** hand over full control; instead, they preserve their own agency in:
    - software design
    - implementation choices
  - This control is driven by concern for **fundamental software quality attributes**.
  - Experienced developers use their expertise to **manage and constrain agent behavior**.
  - Overall sentiment is **positive**, because developers believe they can compensate for agents’ limitations.

- **Implications**
  - The study suggests that established **software development best practices** remain important when using agents effectively.
  - It points toward:
    - better **agentic interfaces**
    - clearer **guidelines for agent use**
  - It also helps identify **which tasks may be suitable** for agents versus those requiring strong human control.

- **What this abstract does and does not tell you**
  - It clearly states the study’s framing, methods, and conclusions.
  - It does **not** provide detailed results, task breakdowns, or specific examples of control strategies in the abstract.
