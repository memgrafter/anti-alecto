### TL;DR
SMITH is a multi-agent system for the GAIA benchmark that combines sandbox-based tool creation, hierarchical memory for cross-task experience sharing, and difficulty-based curriculum learning to reach 81.8% Pass@1, outperforming prior baselines like Alita and Memento.

### Key Quote
“SMITH introduces a unified cognitive architecture that integrates dynamic tool creation with cross-task experience sharing through hierarchical memory organization.”

### Summary
- **What this is**
  - A digest of arXiv paper **2512.11303**, titled **“Unifying Dynamic Tool Creation and Cross-Task Experience Sharing through Cognitive Memory Architecture.”**
  - The digest frames the paper as proposing a unified **cognitive architecture** for agentic task solving.

- **Core idea**
  - SMITH combines three capabilities:
    - **Dynamic tool creation** via iterative code generation and debugging in a **sandbox environment**
    - **Cross-task experience sharing** via **hierarchical memory** and semantic retrieval
    - **Curriculum learning** via **agent-ensemble difficulty re-estimation** to order tasks for better transfer

- **Architecture**
  - Uses a **multi-agent workflow**:
    - **Planner** decomposes tasks
    - **Developer** generates code/tools
    - **Tester** validates in sandbox
  - Memory is organized into:
    - **Procedural memory**
    - **Semantic memory**
    - **Episodic memory**
  - Retrieval is **dense-sparse hybrid**, using:
    - **text-embedding-3-large**
    - **Splade**
    - **Reciprocal Rank Fusion**
  - The developer retrieves **n = 6** memories; planner retrieves **m = 4** memories.

- **How tool creation works**
  - Code is generated, executed in a sandbox, and revised based on structured feedback.
  - The loop continues until successful execution; successful trajectories are stored as reusable **tool memory episodes**.
  - This is presented as a way to create new capabilities without predefining every tool.

- **How experience sharing works**
  - Past successful episodes are abstracted into embeddings and stored.
  - At inference, semantically similar memories are retrieved and appended to the policy context.
  - The paper assumes that semantically similar tasks share transferable solution structure.

- **Curriculum learning**
  - Tasks are ordered using difficulty estimates produced by two proxy agents:
    - **Plan-Execute** (high bias)
    - **ReAct** (high variance)
  - Their consensus is used to re-estimate difficulty and create a curriculum that improves transfer.
  - The digest says this matters because task ordering affects memory reuse and cumulative performance.

- **Results on GAIA**
  - **81.8% Pass@1** overall
  - Breakdown:
    - **Level 1: 94.3%**
    - **Level 2: 80.2%**
    - **Level 3: 61.5%**
  - Baseline comparison:
    - **Alita: 75.2%**
    - **Memento: 70.9%**
  - Ablation:
    - Removing curriculum learning drops performance by **10.3%** (**81.8% → 71.5%**)
  - **3-path sampling** improves Pass@1 by **+3%** but triples inference cost.

- **Mechanistic claims**
  - The digest highlights three mechanisms:
    - **Sandbox debugging** as the gradient-like signal for tool refinement
    - **Semantic retrieval** as the mechanism for cross-task transfer
    - **Difficulty re-estimation** as the mechanism for curriculum optimization
  - Each mechanism is described with assumptions and failure conditions, such as:
    - feedback becoming uninformative
    - embedding similarity failing to capture functional equivalence
    - proxy agents correlating in failure

- **Limitations and open questions**
  - The digest notes several unresolved issues:
    - Long-term drift toward **model-specific biases** as self-generated memory dominates
    - Potential value of using **failed trajectories** as negative samples
    - Unclear generalization beyond **GAIA**
  - It also flags practical limitations:
    - Full procedural prompts unavailable
    - Semantic similarity assumption not directly validated
    - Curriculum approach may not generalize beyond the benchmark
