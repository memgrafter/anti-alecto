### TL;DR
Codex’s “Speed” page explains Fast mode for GPT-5.4—1.5x faster output at 2x credit cost—and contrasts it with the separate, near-instant GPT-5.3-Codex-Spark model.

### Key Quote
“Fast mode is currently supported on GPT-5.4. When enabled, speed is increased by 1.5x and credits are consumed at a 2x rate.”

### Summary
- **Fast mode** is a Codex setting that trades higher credit usage for faster model responses.
- It is **currently supported on GPT-5.4**.
- When enabled:
  - **Speed increases by 1.5x**
  - **Credits are consumed at a 2x rate**
- You can control it in the CLI with:
  - `/fast on`
  - `/fast off`
  - `/fast status`
- You can make it the default in `config.toml` with:
  - `service_tier = "fast"`
  - `[features].fast_mode = true`
- Fast mode is available in:
  - **Codex IDE extension**
  - **Codex CLI**
  - **Codex app** when signed in with ChatGPT
- If you use an **API key**, Codex uses **standard API pricing** instead, and **Fast mode credits are not available**.
- The page also introduces **GPT-5.3-Codex-Spark**:
  - a **separate** model, not just a faster setting
  - optimized for **near-instant, real-time coding iteration**
  - described as **fast but less capable** than GPT-5.4 with Fast mode
  - has its **own usage limits**
- During the **research preview**, Codex-Spark is **only available to ChatGPT Pro subscribers**.
